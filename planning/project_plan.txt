# University Course Reminder Web App
Technologies: HTML, CSS, JavaScript (frontend), Node.js + Express (backend), JSON/Database (data storage)

---

## 1. Project Overview
This web app helps university students stay on top of their weekly academic tasks by automatically reminding them about classes, tutorials, quizzes, and exams. The goal is to reduce time spent checking multiple course outlines.

The system will allow users to:
- Create a profile (name, email, enrolled courses)
- Input data from their course outlines (dates, deadlines)
- Store this data on the server
- Automatically send a weekly summary email with upcoming tasks

Later versions will include:
- Automated data extraction (data mining from course PDFs)
- Calendar integration (Google Calendar, Outlook)
- AI-based reminders or natural-language input (“Add my CS101 midterm Oct 15”)

## 3. Design Process & Development Plan

### Phase 1: Planning
- Identify key user actions: sign up, input course data, view weekly reminders.
- Define basic data model:
  - User: { name, email, courses[] }
  - Course: { courseCode, classDays, assignments[], tests[] }
- Plan server endpoints:
  - POST /api/users -> create user
  - GET /api/users/:id -> get user data
  - POST /api/courses -> add course
  - GET /api/reminders/:userId -> generate weekly summary

---

### Phase 2: Building the Frontend
- Start with simple HTML structure (profile page + dashboard)
- Use CSS to create a clean, readable layout
- Add JavaScript to handle form input, validation, and sending data to the backend using `fetch()`
- Display user’s upcoming tasks dynamically

---

### Phase 3: Building the Backend
- Initialize Node.js project (`npm init`)
- Install Express, Nodemailer (for email), and a lightweight JSON database (like `lowdb` or `fs`)
- Create RESTful routes for users and courses
- Set up email sending (test with a dummy Gmail account)
- Implement a scheduler using `node-cron` to send weekly reminders automatically

---

### Phase 4: Connecting Frontend and Backend
- Test all routes using Postman
- Hook up `fetch()` calls in the frontend to communicate with backend
- Ensure data is being stored and retrieved correctly
- Validate user input and handle server responses

---

### Phase 5: Expansion and Data Mining
- Add PDF upload feature (for course outlines)
- Use a text-extraction library (like `pdf-parse`) to automatically detect important dates
- Improve UI/UX with a dashboard summary view
- Add authentication (JWT or Passport)
- Migrate from JSON storage to MongoDB or PostgreSQL

---

## 4. Optional Tools and Libraries
- **Frontend:** TailwindCSS or Bootstrap for styling
- **Backend:** Express.js for server routes, Nodemailer for email, Node-cron for scheduling
- **Database:** LowDB (beginner) or MongoDB (scalable)
- **Version Control:** Git + GitHub
- **Deployment:** Render, Railway, or Vercel (for free hosting)
